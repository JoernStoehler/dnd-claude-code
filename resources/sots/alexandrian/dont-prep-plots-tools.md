# Don't Prep Plots – Tools, Not Contingencies

*Article by Justin Alexander | June 18th, 2015*

## Core Thesis

The fundamental principle is: "Don't prep plots, prep situations." Rather than creating contingency plans for every possible player action, game masters should develop reusable tools that work across multiple scenarios.

## The Problem with Contingencies

Many GMs understand this lesson at the macro level but repeat the same mistakes microscopically. The author illustrates this using *Eternal Lies*, a published adventure containing extensive passages about hospital escape scenarios. The original text dedicates hundreds of words to potential player tactics: disguises, medicine checks, reassurance techniques, and diversions.

This approach wastes preparation time by attempting to "second-guess your players and developing mutually contradictory material for every possible choice."

## The Toolbox Alternative

Instead of scripting contingencies, the author proposes condensing the hospital escape into a brief toolkit:

- **Layout details** (front door, side entrance, window constraints)
- **NPC awareness levels** (nurses with +2 Awareness collectively)
- **Security positions** (lobby guards, floor guards, Donovan's specific guards)
- **Consequences** (manhunts if boys are removed)

This approach enables GMs to improvise responses to any player tactic—bomb threats, dimension-hopping spells, hostage situations—using the same underlying information.

## Identifying Useful Tools

Two questions determine what deserves preparation:

1. **Where will PCs focus?** What naturally attracts player interest based on scenario clues and narrative direction?

2. **What are NPCs doing independently?** What security exists? What are daily routines? What existing plans might the cult execute regardless of player involvement?

The hospital example demonstrates this: cultists want preventing boy escapes—logically requiring security measures worth documenting.

## Key Distinction

Tools remain permanent fixtures within the scenario's fictional world. Contingencies are reactive branches specifically designed around anticipated player choices. Tools serve multiple approaches; contingencies serve single paths.

The author notes that even lengthy prep works if focused on tools rather than contingencies—length matters less than organization toward utility rather than narrative.
