# Node-Based Scenario Design

**Source:** https://thealexandrian.net/wordpress/7949/roleplaying-games/node-based-scenario-design-part-1-the-plotted-approach
**Author:** Justin Alexander

A comprehensive framework for non-linear adventure design. The series has 9+ parts.

## Core Concept

Instead of linear plot sequences, design adventures as networks of **nodes** — locations, NPCs, or events — connected by **clues** that point players from one node to another. Players navigate the network by discovering and following clues.

## Part 1: The Plotted Approach (Problems)

Traditional adventure design follows linear structure: "A happens, then B happens, then C happens." Every arrow on the plotted flow-chart is a chokepoint. When players deviate, the adventure stalls.

## Part 2 onwards: Node-Based Solutions

Nodes are prepped as **situations** rather than **plots**. Each node contains:
- A situation the PCs encounter
- Clues pointing to other nodes (minimum 3 per the Three Clue Rule)
- NPCs with goals and resources

The GM doesn't predetermine which path players take through the network.

## Connection to SotS

Node-based design maps directly onto GUMSHOE's investigative ability system. Each node contains clues that are automatically found by characters with the right investigative abilities — no roll required. This eliminates the chokepoint problem that plagues traditional mystery adventures.
