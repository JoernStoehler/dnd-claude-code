# Location Conventions

## Structure

### 1. At a Glance box
First thing in the file. Bold key facts, one line each:
- **Type:** [town / village / wilderness / dungeon / building]
- **Size/Scale:** [population, area, or room count]
- **Vibe:** [one sentence capturing the feel]
- **Key Feature:** [what makes it distinctive]

### 2. Overview
2-3 paragraphs: what the location IS, what it's for in the campaign, what players experience when they arrive.

### 3. Notable places
For each:
- Name (bolded)
- One-line description
- Who/what is there
- What players can do or find

For buildings/dungeons: room-by-room reference with consistent format (entry text, furniture, quirks, DCs if applicable).

### 4. NPCs present
Inline brief profiles or pointers to NPC files. For each: name, role, one-line personality, what they know.

### 5. Connections
How this location relates to others: travel times, routes, which hooks involve this location.

## Skimmability

- **"At a Glance" box at the top** — the GM's first 10 seconds
- **Bold key info** within each subsection
- **Tables for lists** (shops, NPCs, room contents) rather than prose
- **Separate "prep reading" from "during play" content** — overview and background are prep; At a Glance, notable places, and NPC briefs are play reference

## File naming

- File: `lowercase-with-dashes.md` (e.g., `millhaven.md`, `weathervane-manor.md`)
- Location: `campaigns/<name>/lore/` for world locations, or within encounter files for adventure-specific locations
